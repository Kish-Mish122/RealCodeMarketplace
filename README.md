# Репозиторий для RealCode (Маркетплейс)
## Как опубликовать свой плагин?
1. Создайте его:
- Через `git clone https://github.com/Kish-Mish122/RealCode .` скопируйте репозиторий в вашу рабочую папку
- Создайте в этой директории вашу папку с плагином (Например: MY_PLUGIN)
- Создайте в нем main.py, где будут все, что вы хотите изменить

Пример кода:
```Python
import tkinter as tk

def init(app):
    app.log("✅ Плагин 'your_plugin' загружен!")

    menubar = app.root['menu']
    if not menubar or not isinstance(menubar, tk.Menu):
        menubar = tk.Menu(app.root)
        app.root.config(menu=menubar)

    plugin_menu = None
    for i in range(menubar.index(tk.END) + 1):
        try:
            if menubar.entrycget(i, 'label') == 'Плагины':
                plugin_menu = menubar.nametowidget(menubar.entrycget(i, 'menu'))
                break
        except:
            continue

    if not plugin_menu:
        plugin_menu = tk.Menu(menubar, tearoff=0)
        menubar.add_cascade(label='Плагины', menu=plugin_menu)

    plugin_menu.add_command(label='Мой плагин', command=lambda: show_dialog(app))

def show_dialog(app):
    window = tk.Toplevel(app.root)
    window.title("Мой плагин")
    window.geometry("350x200")
    window.configure(bg='#f0f0f0')
    window.resizable(False, False)
    window.update_idletasks()
    x = (window.winfo_screenwidth() - 350) // 2
    y = (window.winfo_screenheight() - 200) // 2
    window.geometry(f"+{x}+{y}")

    tk.Label(window, text="🧪 Тестовый плагин",
             font=("Segoe UI", 14, "bold"), bg='#f0f0f0', fg='#333').pack(pady=15)
    tk.Label(window, text="Плагин успешно загружен!",
             font=("Segoe UI", 11), bg='#f0f0f0', fg='#555').pack(pady=10)
    tk.Button(window, text="Закрыть", command=window.destroy,
              bg='#0066cc', fg='white', relief='flat', padx=20, pady=5,
              font=("Segoe UI", 10, "bold"), cursor='hand2').pack(pady=15)
```
- **ВАЖНОЕ УТОЧНЕНИЕ: НЕ ИЗМЕНЯЙТЕ MAIN.PY В ПАПКЕ REALCODE, ТАК КАК ОПУБЛИКОВЫВАТЬ MAIN.PY REALCODE - НЕЛЬЗЯ. ИСПОЛЬЗУЙТЕ ЕГО ДЛЯ `python main.py` ЧТОБЫ ПРОВЕРИТЬ ИЗМЕНЕНИЯ**
- Чтобы посмотреть, применилось ли или все ли правильно, используйте папку с RealCode:
1. Создайте папку `plugins`, если её нет.
2. Создайте подпапку с id вашего плагина (e.g: name_plugin).
3. Скопирйте ваш main.py в эту папку.

- После этого, запустите RealCode через `python main.py`. Проверьте, все ли работает как надо и все ли вам нравиться.

*ВАЖНО: Проверяйте еще и в билде RealCode (компилировать: `python build.py`)*

## ПРАВИЛА ПУБЛИКАЦИИ:
**Главное требование:** Каждый плагин обязан быть безопасным, стабильным и не нарушать авторские права.

*Технические требования:*
- **Чистый код**. Запрещено обфусцировать или шифровать исходный код плагина.
- **Изоляция.** Плагин не должен нарушать работу самой программы RealCode.
- **Ресурсы.** Запрещена чрезмерная нагрузка на процессор и оперативную память.
- **Зависимости.** Все сторонние библиотеки нужно упаковывать внутрь плагина.
- **Версионность.** Обязательно использовать семантическое версионирование (например, 1.0.0).

*Безопасность и конфиденциальность*
- **Данные.** Запрещен скрытый сбор персональных данных пользователей.
- **Сеть.** Любые внешние запросы должны быть четко задекларированы.
- **Вредоносный код.** Спам, майнеры и трояны приведут к вечному бану разработчика.
- **Доступы.** Плагин должен запрашивать только минимально необходимые права.

*Оформление и публикация*
- **Название.** Запрещено использовать чужие бренды и кликбейт.
- **Описание.** Обязательно указать четкие функции плагина на русском языке.
- **Инструкция.** Страница должна содержать примеры использования или команды.
- **Контакты.** Автор обязан оставить ссылку на GitHub или почту для баг-репортов.

*Модерация и поддержка*
- **Проверка.** Каждый плагин проходит ручную премодерацию перед публикацией.
- **Обновления.** Критические баги должны исправляться в течение 5 дней после жалобы.
- **Удаление.** Администрация может удалить плагин при обнаружении уязвимостей или "серьезных" багов.
